## Common Patterns Identified

### Naming Patterns
1. **Single lowercase letter** (a-z): Usually data classes, events, enums, or utility classes
2. **Single uppercase letter** (A-Z): Usually task classes
3. **Two lowercase letters** (aa-zz): Usually task classes
4. **Abstract base classes**: Named with "Base" or "Abstract" in suggested name
5. **Singleton managers**: Named with "Manager" or "Service" suffix

### Task Priority Patterns
- **0x7FFFFFFF (Integer.MAX_VALUE)**: Highest priority, critical tasks (panic teleport, emergency actions)
- **1000+**: Very high priority (banking, portal entry)
- **100-999**: High priority (eating, prayer, combat maintenance)
- **10-99**: Normal priority (core combat tasks)
- **1-9**: Low priority (optional tasks)

### Task Attributes
- **blocking=true**: Task blocks other tasks from running
- **register=true**: Task registers for events (likely EventBus)

### Method Naming Patterns
- Abstract methods that tasks must implement:
  - `h()` in Herblore tasks (likely "handle" or "execute")
  - `ac()` in Zulrah tasks
  - `Q()` in Vorkath tasks
- `run()`: Framework method that validates conditions then calls task-specific method

## Deobfuscation Strategy Recommendations

1. **Start with base classes**: Rename `d`, `am`, `w` to their respective TaskBase classes
2. **Use @TaskDesc annotations**: These provide the exact task names
3. **Follow inheritance**: All tasks in a plugin extend the same base class
4. **Check imports**: RuneLite/Unethicalite API usage reveals functionality
5. **Event classes**: Look for EventBus.post() calls
6. **Enums**: Check for enum constants and field types
7. **Utility classes**: Singletons with @Inject and @Subscribe are managers/services
